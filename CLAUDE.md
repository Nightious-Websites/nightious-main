# Nightious

Astro 5 + Tailwind v4 + TS + GSAP site for `nightious.com`. Optimize for SEO, a11y, responsive, page speed.

## Commands

- `npm run dev` / `build` / `preview` — no unit test runner
- `npm run check:a11y` — axe via Playwright
- `npm run check:media` — asset size budget
- `npm run lhci:local` — build + Lighthouse CI
- Deploys to Vercel on push to `main`; `@vercel/analytics` + speed-insights wired in.

## Workflow

- `git status --short --branch` first; never revert unrelated changes.
- Read files before editing; keep patches narrow and aligned with existing components, tokens, and motion.
- On page add/remove/rename in `src/pages/`: update SEO metadata, canonical/OG, schema/breadcrumbs, nav/footer. `llms.txt` (via `src/pages/llms.txt.ts`), sitemap, robots are generated — don't edit output.

## Map

- Routes `src/pages/` · Layouts `src/layouts/` · Components `src/components/`
- Services `src/content/services/` (helpers: `src/utils/services.ts`, `serviceTheme.ts`)
- Assets: `public/` (raw), `src/assets/` (imported), `src/styles/` (tokens)
- Scripts `scripts/` (a11y, media budget, Lighthouse, screenshot)
- SEO owned by `src/layouts/` (astro-seo) — sitemap/robots configured in `astro.config.mjs`

## Gotchas

- Service entry IDs include `.md`; use `entrySlug()` from `@/utils/services`.
- No `slug` field in service Zod schemas.
- `llms.txt` is generated by `src/pages/llms.txt.ts` from service content; edit the route, not output.
- Hero backgrounds are image + CSS/canvas motion; no video unless asked.
- Image assets via Leonardo MCP (`mcp__leonardo-ai__*`); optimize before importing to `src/assets/`.

## Style

- Dark glass/aurora system; semantic HTML, accessible names, reduced-motion support; copy concise and customer-facing.
